05

Searching & Sorting

Days 5–6 · ~5 hours total
Must Do

What to Learn

  • Linear Search – O(n)
  • Binary Search – O(log n) – sorted arrays only
  • Bubble Sort – understand logic
  • Selection Sort – understand logic
  • Insertion Sort – understand logic
  • sort() from STL – use in all problems
  • Sorting with comparator (custom sort)

Interview Questions

  • Binary search on sorted array (code it)
  • Bubble sort time complexity?
  • Which sort is best for nearly sorted array?
  • First/last occurrence of element using binary search
  • Sort array of 0s, 1s, 2s (Dutch National Flag)

⚠️ Common Mistakes

  • Binary search on unsorted array – won't work
  • Integer overflow in mid: use mid = low + (high-low)/2
  • Saying "I'll implement quicksort" and then fumbling. Just say sort() is O(n log n)
🎯 Practice: Binary search – 10 variations. Sorting – know all 3 sorts by hand

07

OOP – Object-Oriented Programming

Days 7–8 · ~8 hours total – CRITICAL
Must Do

🏛️ Class & Object

Blueprint vs instance. Learn: attributes, methods, access modifiers (public/private/protected). Constructor vs destructor.

class Car {
  public:
    string brand;
    int speed;
    Car(string b, int s) {
      brand = b; speed = s;
    }
    ~Car() { } // destructor
    void display() {
      cout << brand << " " << speed;
    }
};

🔒 Encapsulation

Wrapping data + methods. Private data, public getters/setters. Why? Data protection, validation.

Q: What is encapsulation? Give real example.

A: Hiding internal details. Like ATM – you use it without knowing internal wiring.

👁️ Abstraction

Showing only relevant info. Abstract classes, pure virtual functions. Difference from encapsulation.

Q: Difference between abstraction and encapsulation?

A: Abstraction = WHAT. Encapsulation = HOW it's hidden.

👨‍👦 Inheritance

Types: Single, Multiple, Multilevel, Hierarchical, Hybrid. Use : public BaseClass. Method overriding basics.

class Animal {
  public: void eat() { }
};
class Dog : public Animal {
  public: void bark() { }
};

🎭 Polymorphism

Two types: Compile-time (Function overloading, Operator overloading) and Runtime (Function overriding + virtual keyword).

class Shape {
  public:
    virtual void draw() { }
};
class Circle : public Shape {
  public:
    void draw() override {
      cout << "Drawing Circle";
    }
};

⚠️ Top OOP Mistakes in Interviews

  • Confusing abstraction and encapsulation – memorize: Abstraction = WHAT, Encapsulation = HOW
  • Not knowing virtual keyword – this is asked in 90% interviews
  • Forgetting destructor in classes with dynamic memory
  • Multiple inheritance diamond problem – mention and explain
🎯 Practice: Build 3 projects using OOP: Bank Account class, Shape hierarchy, Student Management
08

STL – Standard Template Library

Days 9–10 · ~6 hours total
Must Do

vector<T>

  • push_back(), pop_back()
  • size(), empty()
  • at(i), front(), back()
  • sort(v.begin(), v.end())

map<K,V>

  • Sorted key-value pairs
  • mp[key] = value
  • mp.find(), mp.count()
  • Iteration: for(auto& p : mp)

unordered_map

  • O(1) average lookup
  • Use for frequency counting
  • Same API as map
  • Use over map when order doesn't matter

set<T>

  • Unique sorted elements
  • insert(), erase(), find()
  • count() – 0 or 1 only
  • Use to remove duplicates

stack<T>

  • LIFO – Last In First Out
  • push(), pop(), top()
  • empty(), size()
  • Use: balanced brackets, undo

queue<T>

  • FIFO – First In First Out
  • push(), pop(), front(), back()
  • empty(), size()
  • Use: BFS traversal
🎯 Practice: Solve 15 LeetCode easy problems using only STL containers. No manual data structures.

Module 3

Aptitude Plan

Many students fail here despite good coding. Don't be one of them.

⚡
Reality: Local companies and service-based companies eliminate 40-60% of candidates in aptitude rounds. Study this even if you're strong at coding.

Percentages

⭐⭐⭐⭐⭐

📅 Daily: 5 questions · 20 min

Key Formulas

  • % increase/decrease = (diff/original) × 100
  • A is what % of B = (A/B) × 100
  • Successive % change: x + y + xy/100

Common Questions

  • Salary increased by 20% then decreased by 20% – net change?
  • A scored 40% of 150. B scored 50% of 200. Who scored more?
  • Price reduced 25%, how much % more can be bought for same money?

Time & Work

⭐⭐⭐⭐⭐

📅 Daily: 5 questions · 20 min

Key Formula

  • Work = Rate × Time
  • If A does work in n days → rate = 1/n per day
  • Combined rate = 1/A + 1/B

Common Questions

  • A does work in 12 days, B in 15 days. Together?
  • A, B, C together in 6 days. A+B in 10. C alone?
  • Pipe fills in 6h, empties in 10h. Together?

Time, Speed & Distance

⭐⭐⭐⭐⭐

📅 Daily: 5 questions · 20 min

Key Formula

  • Speed = Distance / Time
  • Average speed for equal distances = 2ab/(a+b)
  • Relative speed: same dir = |a-b|, opp = a+b

Common Questions

  • Train crosses pole in 10s at 72 km/h. Length?
  • Two trains approach each other. When do they meet?
  • If speed increases by 20%, time saved on fixed distance?

Ratio & Proportion

⭐⭐⭐⭐

📅 Daily: 3 questions · 10 min

Key Concepts

  • a:b = c:d → ad = bc (cross multiplication)
  • Mixing problems, partnership profit splits
  • Dividing amounts in ratio

Profit & Loss

⭐⭐⭐⭐

📅 Daily: 3 questions · 10 min

Key Formulas

  • Profit% = (Profit/CP) × 100
  • SP = CP × (1 + P%/100)
  • Discount = Marked Price – Selling Price

Number System

⭐⭐⭐⭐

📅 Daily: 3 questions · 10 min

Key Concepts

  • HCF, LCM – Euclidean algorithm
  • Divisibility rules (2,3,4,5,6,7,8,9,11)
  • Unit digit patterns, remainders
  • Prime factorization

Average

⭐⭐⭐

📅 Daily: 2 questions · 5 min

Key Formula

  • Average = Sum / Count
  • New average when element added/removed
  • Weighted average

Probability

⭐⭐⭐

📅 Every 3 days: 5 questions

Key Formula

  • P(E) = Favorable / Total outcomes
  • P(A∪B) = P(A) + P(B) – P(A∩B)
  • Dice, cards, coins – know sample spaces

Permutation & Combination

⭐⭐

📅 Every 3 days: 3 questions

Key Formula

  • nPr = n!/(n-r)!
  • nCr = n!/(r!(n-r)!)
  • Arrangements vs selections

Logical Reasoning

⭐⭐⭐⭐⭐

📅 Daily: 10 questions · 20 min

Topics to Cover

  • Syllogism (All A are B type)
  • Blood relations
  • Seating arrangements

Module 4

Communication & HR Interviews

Technical skills get you to HR. Communication skills get you the offer.

⚠️
Brutal Truth: 70% of Tier-3 students get rejected at HR despite passing technical. Nervousness, poor English, and canned answers are the killers. Start practicing Day 1 — talk out loud.

📅 Daily Communication Improvement (15–20 minutes)

Morning (5 min) Read 1 article from Times of India / Hindu aloud
Evening (10 min) Record yourself answering 1 HR question. Listen back. Improve.
Weekend (20 min) Mock HR interview with a friend/peer. Swap roles.
Q1

Tell me about yourself

FRAMEWORK: PRESENT → PAST → FUTURE

Sample Answer

"I am [Name], a final-year B.Tech CSE student from [College]. Currently, I am strong in C++, OOP, and SQL, and have built [1-2 projects]. Before college, I developed an interest in problem-solving through competitive events. Going forward, I want to start my career as a software developer where I can contribute to real products and grow my technical skills. I am a quick learner and I work well under pressure."

⚠️ Don't Do This

  • Don't read your resume aloud – add value beyond resume
  • Don't say "I am a hardworking person who is passionate about…" – too generic
  • Keep it under 90 seconds
Q2

Why should we hire you?

FRAMEWORK: SKILL + PROOF + VALUE ADD

Sample Answer

"I have the core technical skills you're looking for — C++, SQL, and web development. I've applied these in [project name] where I built [specific feature]. Beyond that, I'm someone who genuinely troubleshoots problems rather than giving up — I spent 3 days debugging a database issue in my project. I'm also very open to learning new tools quickly. I believe I can contribute from day one and grow with your team."

Q3

Strengths & Weaknesses

FRAMEWORK: REAL STRENGTH + EXAMPLE · REAL WEAKNESS + IMPROVEMENT STEP

Strength Example

"My biggest strength is persistence. When I started learning C++, I had no prior experience, but I completed [X] programs in 2 weeks by studying consistently. I don't give up when things get hard."

Weakness Example

"I tend to over-think problems before starting. I've been working on this by setting a 10-minute rule — if I haven't started in 10 minutes, I just write the brute force and iterate."

⚠️ Avoid These Weakness Answers

  • "I work too hard" – interviewer knows it's fake
  • "I'm a perfectionist" – overused and unbelievable
  • "I have no weaknesses" – shows lack of self-awareness
Q4

Why this company?

FRAMEWORK: RESEARCH → ALIGN → CONTRIBUTE

How to Prepare

Before every interview, spend 15 minutes on the company website. Learn: what they do, their main product, their tech stack if visible, recent news.

Sample Answer

"I researched [Company] and I see that you work on [specific product/domain]. This aligns with my interest in [related area]. I also noticed your team works with [tech they use], which I have been building skills in. I believe this role will give me real-world experience in an environment where I can make meaningful contributions."

Q5

5-Year Goals

FRAMEWORK: SHORT-TERM SKILL → MID-TERM ROLE → LONG-TERM IMPACT

Sample Answer

"In the next 1–2 years, I want to become proficient in the company's tech stack and deliver reliable features independently. In 3–5 years, I'd like to take on technical leadership for small projects or mentor junior developers. My long-term goal is to become a senior software engineer who can architect scalable systems. I see [Company] as the right place to build that foundation."

Q6

Explain Your Project

FRAMEWORK: PROBLEM → SOLUTION → TECH → CHALLENGE → OUTCOME

Template

"I built [Project Name] to solve [specific problem]. The solution was [what it does in 1 sentence]. I used [tech stack]. The biggest challenge was [specific technical issue] — I resolved it by [your approach]. The final product [result — worked, was deployed, received feedback]."

Example

"I built a Student Attendance System to replace paper-based tracking. It allows teachers to mark attendance and generate reports. I used HTML/CSS/JS for frontend and MySQL for the database. The biggest challenge was designing the database schema for multiple classes and sections — I solved it with a normalized 3-table structure. The final system reduced attendance reporting time by 80%."

Q7

A Challenge You Faced

FRAMEWORK: STAR (Situation · Task · Action · Result)

Sample Answer

"During my final project, 3 days before the deadline, I discovered a critical bug in my login system that allowed unauthorized access. [S] The task was to fix it without breaking existing features. [T] I stayed up two nights, read about session management, and rewrote the authentication module. [A] The project submitted successfully and the security issue was completely resolved. [R] It taught me to always allocate buffer time for testing."

Q8

Explain a Technical Concept Simply

FRAMEWORK: ANALOGY → TECHNICAL DEFINITION → REAL USE CASE

Example: "What is OOP?"

"Think of a blueprint for a house — that's a class. When you actually build a house from that blueprint, that's an object. OOP is a way of writing code where you organize data and behavior together in these blueprints. It helps in real projects because instead of writing the same code again and again, you create a class once — like a Customer class — and reuse it everywhere in your application."

🚫 Top Communication Mistakes in Interviews

❌
Starting with "So…" or "Basically…"

Filler words make you sound unsure. Start with a direct sentence.

❌
Looking away when nervous

Maintain soft eye contact. Looking at camera/interviewer = confidence.

❌
Saying "I don't know" and stopping

Say: "I haven't worked on this specifically, but my understanding is…" Show thinking.

❌
Memorized answers that sound robotic

Practice the framework, not the exact words. Conversation beats recitation.

❌
Not asking any questions at the end

Always ask: "What does the first 90 days look like for this role?"

❌
Speaking too fast due to nerves

Slow down. Pause between sentences. Silence is better than rambling.

Core CS Subjects

DBMS, SQL, OS, CN — know these cold. They're asked in almost every interview.

🗄️ DBMS

Priority: Very High

Must-Know Concepts

ACID properties Normalization (1NF–3NF + BCNF) ER Diagrams Keys (Primary, Foreign, Candidate, Super) Transactions Indexing Joins (Inner, Left, Right, Full) Views vs Tables Stored Procedures Triggers

🔥 Most Asked Questions

  1. What are ACID properties? Explain each with example.
  2. What is normalization? Explain 1NF, 2NF, 3NF with example.
  3. Difference between Primary Key and Unique Key?
  4. What is a foreign key? What happens on delete?
  5. Explain different types of JOINs.
  6. What is a transaction? What is a deadlock?
  7. Difference between DELETE, TRUNCATE, DROP?
  8. What is indexing? How does it speed up queries?
  9. Difference between clustered and non-clustered index?
  10. What is a View? Can you update through a view?

📋 Revision Strategy

Day 11–13: Read concepts → make 1-page cheat sheet. Day 20–22: Revise cheat sheet. Day 28–30: Answer all questions aloud without notes.

💾 SQL

Priority: Critical — Written + Verbal

Must-Know Commands

DDL
CREATE TABLE, ALTER TABLE, DROP TABLE, TRUNCATE
DML
SELECT, INSERT INTO, UPDATE SET, DELETE FROM
Clauses
WHERE, GROUP BY, HAVING, ORDER BY, LIMIT
Functions
COUNT, SUM, AVG, MAX, MIN, DISTINCT
JOINs
INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN
Subqueries
SELECT inside WHERE, FROM, HAVING

🔥 Most Asked SQL Queries

Find the second highest salary

SELECT MAX(salary) FROM employees
WHERE salary < (SELECT MAX(salary) FROM employees);

Count employees per department

SELECT dept, COUNT(*) as count
FROM employees GROUP BY dept;

Find duplicate records

SELECT name, COUNT(*) FROM employees
GROUP BY name HAVING COUNT(*) > 1;

📋 Revision Strategy

Practice on SQLZoo.net or HackerRank SQL section. Do 30 minutes of SQL coding daily from Day 11 onward.

🔥 Most Asked Questions

  1. Difference between process and thread?
  2. What are the four conditions for deadlock?
  3. Explain Round Robin scheduling with an example.
  4. What is virtual memory? How does it work?
  5. Difference between semaphore and mutex?
  6. What is thrashing?
  7. Explain paging vs segmentation.
  8. What is context switching?

🌐 Computer Networks

Priority: High

Must-Know Concepts

OSI Model (7 layers) TCP vs UDP HTTP vs HTTPS DNS IP Addressing (IPv4, IPv6) Subnetting basics TCP 3-Way Handshake ARP, DHCP, ICMP Routing algorithms Firewalls

🔥 Most Asked Questions

  1. Explain OSI model layers with functions of each.
  2. Difference between TCP and UDP? When to use each?
  3. What happens when you type google.com in browser?
  4. Explain TCP 3-way handshake.
  5. Difference between HTTP and HTTPS?
  6. What is DNS and how does it work?
  7. Difference between hub, switch, and router?
  8. What is a MAC address vs IP address?

Resume Guidance

Your resume gets you the interview. Make it count in under 6 seconds of scanning.

✅ Skills to Keep

  • C++ (if you've practiced this plan)
  • Data Structures & Algorithms
  • SQL / MySQL
  • HTML, CSS, JavaScript (if you have projects)
  • Git & GitHub (learn basics – 1 day)
  • DBMS fundamentals
  • Operating System concepts
  • Problem Solving (LeetCode / GFG)

🚫 Skills to Remove

  • Any tool you can't explain in detail
  • "MS Office" – everyone has this, wastes space
  • Technologies you've only "heard of"
  • "Photoshop" unless design role
  • Rating your own skills (Expert / Intermediate) – never do this
  • Tools from college lab assignments you barely remember

📁 Projects to Include

  • Include 2–3 projects maximum
  • At least 1 with a database (SQL-based)
  • At least 1 that's deployed (GitHub Pages / Netlify)
  • Good examples: Library Management, Expense Tracker, Student Portal, Quiz App, Attendance System
  • For each project write: What it does, Tech stack, Your specific contribution, Link (GitHub)

📝 ATS-Friendly Rules

  • Use simple single-column layout (no tables, no columns for ATS)

Company-Wise Strategy

Different companies test you differently. Prepare specifically, not generically.

🏘️

Local / Small Companies

Visiting your college first — easiest entry

Interview Pattern

  • Written aptitude (30–60 min)
  • 1–2 technical rounds (30 min each)
  • 1 HR round
  • Coding might be pen-paper or verbal

Difficulty Level

Low–Medium. Questions are basic to intermediate. Focus on basics and communication.

Frequently Asked Topics

Basic C++ / OOP SQL queries (basic) Array, String problems HR questions Aptitude (arithmetic) Project explanation
💡 Key Strategy: Be confident, speak clearly, and have 2 solid projects ready to demo. They value attitude and learning ability more than advanced DSA.
🚀

Startups

Move fast, broader skills expected

Interview Pattern

  • Often no written aptitude
  • 1–2 technical rounds (practical focus)
  • Take-home assignment common
  • Culture fit interview
  • GitHub/portfolio review

Difficulty Level

Medium. More practical than theoretical. Build things, not just answer questions.

Frequently Asked Topics

Project deep-dives Frontend/Backend (role-specific) REST APIs concept Git & GitHub usage Problem solving (easy-medium) System design (very basic) Learning ability
💡 Key Strategy: Have real, working projects on GitHub. Know your project's architecture inside out. Show genuine curiosity about their product.
🏢

Service-Based Companies

TCS, Infosys, Wipro, Cognizant pattern

Interview Pattern

  • Online aptitude test (60–90 min)
  • Coding round (2–3 questions)
  • Technical interview (1–2 rounds)
  • HR interview
  • Group discussion (some companies)

Difficulty Level

Medium. Aptitude is the biggest filter. Technical is moderate — basics tested well.

Frequently Asked Topics

Aptitude (heavy) Logical Reasoning Verbal English Core CS (DBMS, OS, CN) OOP concepts Basic coding (arrays, strings) HR (behavioral)
💡 Key Strategy: Aptitude preparation is non-negotiable here. Practice 40+ questions daily. For coding, focus on easy–medium array/string problems. Polish your English communication.
